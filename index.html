Web API

There's a lot to love about building API's in .Net, but among my favorites is the magic that happens at the controller. It's easy to take the ApiController base class for granted, but if you've ever made a NodeJS API you'll know that .Net takes care of a lot of tedium for you. I love that all you have to do is add an attribute to your controller method (HttpGet, HttpPost, HttpDelete, HttpPut, etc...) and the framework takes care of routing for you. You can, of course, customize the routing by supplying parameters into the previously mentioned attributes, but that's optional. It's so nice to then add Swagger on top to give your API a user-friendly interface, and so much of that is magical as well. I love opening up the startup.cs and whatever other startup/config files are in the API to peek at the what Auth the API uses, what dependency injection framework it's using, is CORS enabled, is it strict on JWT's, is it using JWT's at all? All the cool stuff is in the startup.cs and its friend files.

Entity Framework (and Linq)

Back in the my dark ages where I was building personal websites with PHP on the backend (very very long time ago), I was getting and writing to and from my MySql databases uses hard-coded statements as strings right in my PHP code. I'm glad I went through that because it helps me appreciate Entity Framework, and ORMs in general I suppose. My first experience working with databases professionally was at my time at Dell Software. I'd just finished my 18 month internship at Intel and was still very green. My team at Dell was awesome and introduced me to so many aspects of C# and .Net. Entity and Linq are simply amazing and work so beautifully together. I love that I can setup Entity using the onModelCreating method to rename columns to abstract from myself to sometimes really bad names people give to columns. Another really cool thing about the onModelCreating method is that you can pre-populate your DB with dummy data! No need to have a separate SQL script for seeding data, do it all in code using our favorite language, C#. After getting the data there's nothing better than calling FirstOrDefault(), or Select() to convert all your data into DTO's, or calling Include() to fetch relational data. Such an exciting combination of .Net tools.

Dependency Injection (and Moq)

Unity was great to work with, but now .Net Core comes with dependency injection built-in. The benefits of using DI are obvious if you've been developing for any decent amount of time, and I love that the mindset is such that DI should be there from the start of a project. When working on a WebAPI, most of your registrations will look like this:

services.AddScoped<ICarpoolDetails, CarpoolDetails>();

So clean and simple, yet very customize and powerful if you need it to be.
